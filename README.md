# ⌚ eWatch Landing Page (Figma to Code)

A fast-paced implementation of a modern product landing page based on a Figma design.  
Built using **pure HTML & CSS** with a focus on **speed, layout accuracy, and clean structure**.

---

## 🚀 Overview

This project was completed as a **speed & efficiency test**, translating a static Figma design into a structured and visually accurate layout using only:

- HTML5
- CSS3 (Flexbox + Absolute Positioning)

No frameworks or libraries were used — only core frontend fundamentals.

---

## 🎯 Key Features

- ⚡ Clean **two-section layout** (content + image composition)
- 🎨 Modern UI with consistent spacing and typography
- 🧩 Structured and readable HTML hierarchy
- 🖼️ Complex **layered image composition**
- ✨ Smooth CTA hover interactions

---

## 🧱 Structure

- **main-wrapper**
  - **content-wrapper**
    - header-container
      - logo-container
      - text-content-container
    - list-wrapper
  - **images-main-wrapper**
    - image-text-content-wrapper
    - main-image-container
    - float image containers
    - decorative stars
---

## 🖼️ Image Composition

The right-side visuals were built using:

- Absolute positioning for precision
- Z-index layering for depth
- Floating background shapes
- Decorative stars and spark elements
- Overlay badges

This recreates the visual hierarchy and depth from the original design.

---

## 🎨 Styling Approach

- Flexbox used for layout alignment
- Absolute positioning for controlled overlays
- Consistent spacing and sizing system
- Subtle shadows and hover effects for interactivity

---

## 🧠 Challenges Solved

- Structuring layout with **separate content and image sections**
- Recreating **layered UI elements accurately**
- Maintaining **visual hierarchy (primary vs decorative elements)**
- Fine-tuning positioning for design consistency

---

## 💡 Focus Areas

- Speed and efficiency under time constraints
- Clean and maintainable CSS
- Accurate visual reproduction
- Logical component separation

---

## 📌 Notes

- Designed for desktop layout (as per requirement)
- No external dependencies used
- Easily extendable for responsiveness
